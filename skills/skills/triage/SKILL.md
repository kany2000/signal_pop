# Triage

通过一组小型 triage roles state machine，在项目 issue tracker 中移动 issues。

## 原则

Triage 期间发布到 issue tracker 的每条 comment 或 issue **必须**以 disclaimer 开头：
```
> *This was generated by AI during triage.*
```

## Roles

### Category roles
- **bug** — 某个东西坏了
- **enhancement** — 新功能或改进

### State roles
- **needs-triage** — maintainer 需要评估
- **needs-reproduction** — 需要更多信息来复现
- **ready-for-agent** — 问题清晰，可自动分配给 agent
- **ready-for-dev** — 问题清晰，分配给开发者
- **blocked** — 被外部依赖阻塞

## Flow

1. **读取 issue**，理解内容和上下文
2. **分类**：bug vs enhancement
3. **评估清晰度**：
   - 如果信息不足 → `needs-reproduction`，comment 询问具体信息
   - 如果清晰且可执行 → `ready-for-agent` 或 `ready-for-dev`
   - 如果依赖外部 → `blocked`，comment 说明阻塞原因
4. **对 `ready-for-agent`**：确保 issue 包含：
   - 清晰的 problem statement
   - 复现步骤（bug）或验收标准（enhancement）
   - 相关文件/代码路径
5. **对 enhancement**：检查是否有 PRD 或 design doc 引用
